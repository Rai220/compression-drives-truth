## Стратегия ревизии

Текущая рукопись заявляет сильный тезис ("сжатие ведёт к истине"), тогда как эксперименты подтверждают более узкое утверждение:

> Давление сжатия в моделях с ограниченной ёмкостью предпочитает наиболее консистентную и сжимаемую структуру в данных. Истина получает преимущество только тогда, когда ложные альтернативы требуют дополнительной длины описания или содержат несогласованности.

Ревизия:
1. Исправить внутренние противоречия и числовые ошибки
2. Переформулировать тезис: **консистентность vs истина**
3. Усилить экспериментальный дизайн
4. Добавить целевые эксперименты, демонстрирующие границы эффекта

## P0 — Целостность результатов (блокирует всё остальное)

### 1. Автоматическая сборка таблиц результатов

Написать `scripts/collect_results.py`, который:
- парсит все логи обучения/оценки
- создаёт `results_master.csv`
- генерирует Markdown-таблицы для статьи

**Критерий:** все числа в статье генерируются автоматически одной командой.

### 2. Исправить знаковые несогласованности ΔLoss

Единая конвенция:
```
ΔLoss = Loss(incorrect) − Loss(correct)
```
- ΔLoss > 0 → модель предпочитает correct
- ΔLoss < 0 → модель предпочитает incorrect

**Критерий:** все таблицы следуют одной конвенции, нет противоречий между ΔLoss и описанием предпочтения.

### 3. Пересчёт статистики

- Явно описать статистический тест (биномиальный тест по seeds + bootstrap CI)
- Пересчитать p-values из сырых результатов
- Воспроизводимый статистический вывод

### 4. Аудит библиографии

Создать `docs/refs_audit.md`. Проверить каждую ссылку:

## P1 — Переформулировка тезиса и переписывание статьи

### 5. Сменить заголовок

Варианты:
- "When Does Compression Favor Truth?"
- "Compression Prefers Consistency"
- "Truth, Consistency, and Compression in Language Models"

**Критерий:** заголовок не подразумевает универсальное стремление к истине.

### 6. Переписать аннотацию

Ключевые пункты:
- модели сжимают **текст**, а не реальность
- преимущество истины зависит от структуры корпуса
- когерентные ложные системы могут сжиматься столь же хорошо

### 7. Переписать введение

Три явных оговорки:
1. Исследование анализирует **текстовые корпуса**, а не внешний мир
2. Модели конечной ёмкости могут выучивать **частоту раньше истины**
3. Преимущество сжатия зависит от **длины описания**

### 8. Разделить теоретические концепции

Чётко различать:
- кросс-энтропию и длину кода
- lossless-сжатие последовательностей токенов
- Minimum Description Length (MDL)
- индукцию Соломонова

**Критерий:** нет утверждений, что MDL гарантирует истину; Соломонов — только как интуиция.

### 9. Уменьшить акцент на Поппере

Поппер — только в разделе Discussion как интерпретативная аналогия.
Убрать сильные утверждения, связывающие эксперименты с фальсифицируемостью.

### 10. Убрать риторические аналогии из основных экспериментов

Заменить "гомеопатия", "астрология" на нейтральные названия условий:
- ad hoc exception condition
- systematic correction condition
- underspecified prediction condition

Примеры из реальных псевдонаук — только в Discussion.

## P2 — Улучшение экспериментального дизайна

### 11. Внедрить парную оценку (paired evaluation)

Текущая оценка: случайные окна из текста корпуса.

Замена: для каждой задачи — одинаковый prompt, два завершения (correct / incorrect).

Метрики:
- NLL полного решения
- NLL только ответа
- accuracy финального ответа

**Критерий:** каждое сравнение использует идентичные промпты.

### 12. Контроль длины последовательностей

Текущая проблема: корпуса условий C/D/E имеют разную длину (loss ~0.24-0.25 vs baseline ~0.14).

Два варианта:
1. Выровнять длину последовательностей (padding / нормализация)
2. Обучать до сопоставимой сходимости

**Критерий:** сравнения не confounded различиями длины.

### 13. Оценка на полном датасете

Заменить случайное семплирование батчей на детерминированную оценку по всему тестовому набору.

Добавить:
- corpus-level NLL
- bootstrap CI
- статистику по seeds

### 14. Факторный анализ переменных

Явно разделить экспериментальные факторы:
- истинность (truth value)
- частота (frequency)
- когерентность правил (rule coherence)
- рассогласование с наблюдениями (observation mismatch)
- overhead коррекции (correction overhead)

**Критерий:** каждый эксперимент изолирует один первичный фактор.

## P3 — Новые эксперименты

### Эксперимент A: Парный арифметический тест

**Цель:** убрать confound распределения промптов.

- Идентичные промпты, correct vs incorrect completions
- Ожидание: случайные ошибки → выше NLL; когерентные ложные системы → слабее эффект

### Эксперимент B: Частота vs истина

**Цель:** проверить, побеждает ли популярность истину.

- Две когерентные системы, варьируем пропорцию большинства
- Определить взаимодействие: истина × частота × когерентность

### Эксперимент C: Тест фальсифицируемости с контролем длины

**Цель:** проверить, сохраняется ли эффект overhead коррекции после контроля длины.

- Пересоздать Exp 3 с идентичным token budget
- Если эффект сохраняется → сильный результат
- Если исчезает → предыдущий результат — артефакт

### Эксперимент D: Масштабирование по ёмкости (scaling)

Размеры: `small` (11M), `medium` (26M), `large` (86M).

**Цель:** проверить, как truth preference меняется с ёмкостью.

- Гипотеза "обратной U-кривой": слишком маленькие не могут выучить правила, средние → максимальный bias, большие запоминают всё
- **Критерий:** монотонный или интерпретируемый тренд

### Эксперимент E: Синтетический мир (естественный язык)

Построить симулированный мир с известными правилами (температура → замерзание, свет → рост, давление → кипение).

- true narrator vs false narrator vs false narrator + correction rules
- Модели: 25M–350M, 3–5 seeds
- **Критерий:** known ground truth сохранён
