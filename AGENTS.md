## Окружение

Python venv находится в `.venv/`. Активация:

```bash
source .venv/bin/activate
```

Все скрипты запускать после активации venv. Примеры:

```bash
# Генерация корпуса
python data/generate_math.py --n 200000 --correct-ratio 0.5 --output data/corpus/train_mixed_50_50.txt

# Обучение
python training/train.py --corpus data/corpus/train_mixed_50_50.txt --model tiny --steps 5000 --seed 42 --output results/mixed_50_50_tiny_seed42

# Оценка перплексии
python training/eval_perplexity.py --model-size tiny --weights results/mixed_50_50_tiny_seed42/model_final.npz --tokenizer results/mixed_50_50_tiny_seed42/tokenizer.json --test-correct data/corpus/test_correct.txt --test-incorrect data/corpus/test_incorrect.txt

# Генерация текста
python training/generate.py --model-size tiny --weights results/mixed_50_50_tiny_seed42/model_final.npz --tokenizer results/mixed_50_50_tiny_seed42/tokenizer.json --prompt "Problem: " --max-tokens 200 --temperature 0.3 --n 5

# Построение графиков
python analysis/plot_results.py
```

## Стиль написания статьи

Итоговый текст в `paper_draft_ru.md` и `paper_draft_en.md` должен соответствовать академическому стилю. Соблюдай следующие требования:

### Академический стиль
- Используй безличные и пассивные конструкции: «было показано», «результаты свидетельствуют», а не «мы увидели» или «модель решила».
- Количественные утверждения всегда подкрепляй числами, p-значениями и доверительными интервалами.
- Избегай категоричных оценок без оснований («доказывает», «безусловно»). Предпочитай осторожные формулировки: «согласуется с гипотезой», «позволяет предположить».
- Структура абзаца: тезис → обоснование/данные → вывод. Не начинай абзац с данных без контекста.
- Ссылки на литературу оформляй в едином стиле (автор, год).

### Защита от «сгенерированного» текста
- **Запрещённые слова и обороты** (маркеры машинного текста): «важно отметить», «стоит подчеркнуть», «интересно, что», «следует отметить», «примечательно», «в заключение хотелось бы», «данный подход», «в рамках данного исследования», «является ключевым», «играет важную роль», «notably», «it is worth noting», «importantly», «it should be noted», «crucial», «pivotal», «landscape», «delve into», «shed light on», «pave the way». Если замечаешь эти обороты — перепиши предложение.
- **Не начинай** смежные предложения или абзацы с одинаковой синтаксической конструкции. Чередуй длину предложений: короткие (5–10 слов) с длинными (20–30 слов).
- **Избегай перечислений ради перечислений.** Списки допустимы только для структурированных данных (условия эксперимента, гиперпараметры). Выводы и аргументацию излагай связным текстом.
- **Не используй** шаблонные переходы между разделами («In this section, we will discuss...», «В данном разделе рассматривается...»). Переходи к сути сразу.
- **Каждое предложение должно нести новую информацию.** Удаляй предложения, которые лишь пересказывают предыдущее другими словами.
- При сомнении — перечитай абзац и спроси себя: «Мог бы это написать исследователь от руки?» Если нет — перепиши.
